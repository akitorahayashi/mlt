# `defaults` で何ができるか

まずは`defaults` で何ができるかです。

すべてのOS・アプリケーション設定にアクセスし、読み書きを行うための標準コマンドラインツールが`defaults`です。
`defaults` はmacOSに標準搭載されている、Apple純正のツールです。

`defaults`を理解して使えるようになれば、macbookやmac mini、あるいは複数の端末を持つ人は、例えば、defaults コマンドをまとめたスクリプトを編集して、それぞれで実行するだけで、他の全ての端末で統一した設定を管理することができます。一方で、一つの端末の設定をいくつか変えたときに、いちいち、複数の端末で設定アプリから該当する設定を検索するのは面倒ですよね。

これは新しいmacOSの端末を入手した場合や、macOSに溜まった作業ログ等がストレージを圧迫した際に、工場出荷状態に戻す処置であるクリーンインストールをした際にも有効で、自分好みの設定項目を適用するコマンドを集めたスクリプトさえあれば、実行するだけで全ての設定が復元されます。

---

# ドメイン

`defaults`コマンドでは、設定を管理する単位をドメインと呼びます。

具体例は、Dock の設定を持つ `com.apple.dock`、Finder の設定を持つ `com.apple.finder`、ユーザ領域の全体共通の設定を扱う `NSGlobalDomain` などがあります。

---

# 設定ファイルの実体

これらのドメインは、物理的には`.plist`(プロパティリスト) という形式のファイルとしてストレージ上に保存されています。
これはApple 系のOSやアプリで設定・メタデータ・構成情報を保存するためによく使われるファイル形式です。

設定ファイルの主な保存場所はスライドの通りで、重要な点は、ユーザレベルの設定とシステム全体の設定、アプリの設定が分かれていること、システム全体の設定を変更する際には sudo が必要なことです。


<!-- * ユーザーレベルの設定: `~/Library/Preferences/`
    * 現在ログインしているユーザー専用の設定が保存されます。通常のアプリケーションのカスタマイズの大半はここに該当します。
* システム全体の設定: `/Library/Preferences/`
    * すべてのユーザーに影響を与えるシステムレベルの設定が保存されます。この領域の書き換えには管理者権限（sudo）が必要です。
* サンドボックス化されたアプリの設定: `~/Library/Containers/[ドメイン名]/Data/Library/Preferences/`
    * App Storeからダウンロードしたアプリなど、セキュリティのためにサンドボックス化されているアプリは、隔離された専用のディレクトリ内に設定ファイルを保持します。 -->

このplistという形式のファイルですが、iPhoneアプリを開発をする上でもそのアプリの設定ファイルとして存在していて、この場合などは人間が直接読み書きできる xml の形式ですが、OSに保存されている設定項目の文脈の plist は、OSのパフォーマンス向上と起動時間の短縮、およびストレージ容量の節約を目的として、バイナリ形式で保存されます。

バイナリ形式の`.plist`の中身は直接編集することはやってはいけないやり方で、編集する際には`defaults`を使います。

# 直接編集が危ない理由

ファイルを直接編集してはいけない理由は、macOS が`cfprefsd`というデーモンを使って`.plist`の内容を「メモリ上にキャッシュ」しているからです。

デーモンとは、一言で言うとユーザーの目に触れないところで、働き続けるバックグラウンド・プロセスのことです

アプリやOS は、ディスク上のファイルではなく、メモリ上のキャッシュから設定を読み込んでいます。もしテキストエディタで`.plist`を直接編集すると、ディスク上のファイルは更新されますが、メモリ上のキャッシュは古いままで、新しい設定が反映されません。

`defaults write` を実行すると、ファイルを書き換えると同時に`cfprefsd`に「キャッシュを無効化せよ」という通知を送信します。その後、アプリが設定を参照するタイミングで、cfprefsd は新しい値をディスクから読み込みます。プロセスを`killall`で再起動するのは、起動時に新しい設定を確実に読ませるためです。

※ 統合開発環境（Xcode）の利用:
Apple公式のIDEであるXcodeをインストールしている場合、`xed`コマンドでファイルを開く(xcodeで開く)と、バイナリ形式のまま表形式で編集・閲覧が可能ですが、キャッシュに反映されず、意図しない動作の原因となるため非推奨です。

# どうやって探すか

次に設定可能な項目を探索する方法ですが、ざっくり言うと、コミュニティやAIを活用して特定した項目に対して `defaults read` を実行して読み取りを試みて、有効な値だとわかる場合に`defaults write`で書き込み、変更を行うのが安全です。
Appleは `defaults`コマンドで設定可能な全項目のリストを今のところは作成されていません。

# 実行方法

では実際にどうやってdefaultsを書いて設定を変更するのかという点について説明します。defaults コマンドは、基本的に以下の構造で指定します。

defaults write [ドメイン名] [キー] -[型] [値]

実践的な実行例をいくつか挙げます。`killall` はプロセスを強制終了し、OS が自動的に再起動させます。この再起動時に、プロセスは cfprefsd から最新の設定キャッシュを読み込み、変更が反映されます。

Dockにあるアプリのアイコンの大きさを変更する
> `defaults write com.apple.dock tilesize -int 30`
> `killall Dock`

Finderのタイトルバーに現在のディレクトリのフルパスを表示する（Boolean型の書き込み）
> `defaults write com.apple.finder _FXShowPosixPathInTitle -bool true`
> `killall Finder`

# シェルスクリプトにまとめる

どのmacOSでも保持しておきたい設定は、1行ずつ手で実行するよりも、シェルスクリプトにまとめておくほうが扱いやすくなります。

たとえば、Dock と Finder の設定をまとめて反映するなら、次のように書けます。

```sh
#!/usr/bin/env bash
set -euo pipefail

defaults write com.apple.dock tilesize -int 30
defaults write com.apple.finder _FXShowPosixPathInTitle -bool true

killall Dock
killall Finder
```

このようにしておくと、Mac を初期化したあとでも、スクリプトを 1 回実行するだけで同じ設定をまとめて戻せます。設定を増やすときも、ファイルに追記するだけで管理できるので、再現性の高いセットアップ手順になります。

# まとめ

`defaults` を使えるようになると、macOS の細かな設定を 設定アプリで毎回探さなくても、コマンドをスクリプトにまとめると短時間で再現できるようになります。

つまり、
- macbook, mac miniなど複数のmacOS、デバイスで統一された設定を利用することができます。

- 新しい Mac やクリーンインストール後でも、同じ設定を短時間で戻せます。

共通化しておきたい設定は、設定アプリでの場所を覚えるよりも `defaults read` と `defaults write` を使う流れにしておくほうが実用的です。
